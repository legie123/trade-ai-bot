Probleme UX Identificate

🔴 Status Page: 15 secțiuni într-o coloană infinită

Pagina /dashboard are 1200 linii de cod, 15 Section-uri collapsibile, toate stacked vertical. User-ul trebuie să scrolleze ~3000px ca să vadă totul. Nu există "dashboard overview" — totul e la același nivel de importanță.

Fix: Grupează în 3 zone: (1) CRITICAL STATUS (top, always visible) — health, kill switch, alert level, equity; (2) OPERATIONS (middle) — terminal, commands, strategies; (3) DEEP DIVE (bottom, collapsed by default) — system resources, diagnostics, history.

🔴 Butoane de comandă fără ierarhie

Command Center: 25+ butoane cu aceeași dimensiune, aceeași culoare. Kill Switch ENGAGE e vizual identic cu "Ping Watchdog". Un misclick pe ENGAGE vs DISENGAGE poate costa bani reali.

Fix: Kill Switch = roșu mare, izolat, cu confirmare. Agent commands = albastru. Data collection = gri. Diagnostics = outline. Adaugă confirm() pe orice acțiune distructivă.

🟡 Inconsistență color system între pagini

Cockpit folosește C.cyan/#00e5ff + violet. Arena folosește gold theme. Dashboard folosește un al 3-lea color system propriu (#07080d, #29b6f6). Nu se simte ca aceeași aplicație.

Fix: Un singur color system definit în globals.css, importat peste tot. Permis: accent diferit per pagină, dar backgrounds, borders, text trebuie identice.

🟡 Font sizes prea mici pe label-uri operaționale

Multe label-uri au fontSize: 8px sau 9px. Pe desktop sunt greu de citit; pe mobil sunt imposibile. Minimum recomandat WCAG pentru text operațional: 11px / 0.6875rem.

Fix: Minimum 10px pe desktop, 11px pe mobile. Label-urile de metrici cheie (EQUITY, P&L, WIN RATE) — minim 12px.

🟡 GoldDust canvas pe toată pagina — performance drag

GoldDust.tsx rulează un canvas full-screen cu 40 particule + glow pe requestAnimationFrame. Pe mobile low-end, asta consumă battery și frame budget inutil. Pe desktop cu multiple tabs = memory leak potential.

Fix: (1) Reduce la 20 particule pe mobile. (2) Pause animation când tab-ul nu e vizibil (document.visibilitychange). (3) Disable complet pe prefers-reduced-motion. (4) IntersectionObserver — nu anima dacă nu e vizibil.

✅ CE E BUN

HelpTooltip pe fiecare secțiune — excelent UX educațional. Utilizatorul învață ce face fiecare panel.

Collapsible sections — permite organizare mentală.

FreshnessBadge — indică clar cât de vechi sunt datele. Esențial într-un dashboard de trading.

Command palette (⌘K) — power-user feature premium.

Color coding consistent pe statuse — green/amber/red e intuitiv.

Sidebar cu keyboard shortcuts — profesionist.

2. AUDIT OPERABILITATE

Butoane & Controls

🔴 Kill Switch fără confirmare pe Status Page

Pe /dashboard, Command Center — butonul "ENGAGE KILL SWITCH" se execută cu un singur click, fără confirmare. Pe Cockpit există un pattern de armare (2 click-uri + timeout), dar pe Status page lipsește complet.

Fix: Implementează același pattern de armare pe ambele pagini. Minimum: window.confirm("ENGAGE KILL SWITCH? All trading stops immediately.")

🟡 Butoane de orchestrare hardcodate pe 2 simboluri

"Orchestrate BTCUSDT" și "Orchestrate ETHUSDT" sunt singurele opțiuni. Nu poți alege alt simbol fără a face request manual.

Fix: Adaugă un input field "Custom symbol" + buton "Orchestrate" lângă cele existente. Sau un dropdown cu top 10 active coins din pipeline.

🟡 Filtre terminal limitate

Terminal-ul live are filtre ALL/CMD/ERROR/LOG, dar nu poți filtra după simbol, timestamp, sau nivel de severitate combinat. Într-un terminal cu 500 linii, asta face debugging lent.

Fix: Adaugă search box cu filter live (regex-capable). Adaugă timestamp range picker (last 5m/15m/1h).

✅ BUN: Terminal cu auto-scroll, export JSON/CSV

Export-ul funcțional cu JSON + CSV și suport navigator.share pe mobile e o funcționalitate premium rară. Bine gândit.

Grafice & Vizualizări

🟡 Sparkline-uri prea mici, fără interactive hover

Charts & Trends are 11 mini-carduri cu sparklines de 120x32px. Nu poți hover pentru valori, nu poți zoom, nu poți selecta period. Sunt decorative, nu operaționale.

Fix: (1) Click pe card = expand to full-width chart cu hover tooltips. (2) Adaugă min/max/avg sub fiecare sparkline. (3) Mărește la 160x48px minim.

🟡 BarChart-uri cu font 7-8px pe labels

Label-urile de sub barchart-uri au fontSize: 7. Imposibil de citit pe orice dispozitiv.

Fix: Minimum 10px. Dacă nu încape, folosește tooltip on hover în loc de label permanent.

✅ BUN: AgentStatusHero canvas animation

Synapse pulse e vizual impresionant, se adaptează la starea agentului (viteză, culoare). Excelent feedback vizual live.

Filtre & Sortare

✅ Arena: sortare funcțională pe WIN RATE / PROFIT FAC / MAX DD

Toggle sort cu direcție asc/desc, vizual clar care coloană e activă. Expandable fighter cards cu detalii complete. Bine implementat.

🟡 Lipsă: filter global pe gladiatori (status, arena, live/shadow)

Nu poți filtra rapid doar gladiatorii LIVE, sau doar cei din arena "MAIN". Pe 30+ gladiatori, scrolling-ul e ineficient.

Fix: Adaugă filter pills: ALL | LIVE | SHADOW | ELIMINATED deasupra leaderboard-ului.

3. AUDIT OPERAȚIONALITATE

Health Indicators & Monitoring

✅ EXCELENT: Core Services Strip

STREAM / HEARTBEAT / WATCHDOG / KILL SW / SUPABASE / MODE / ALERT — toate vizibile într-o linie scrollabilă. Asta e cel mai bun element din dashboard. Păstrează-l exact cum e, eventual fă-l sticky sub header.

✅ BUN: Alert bar cu nivel RED/YELLOW/GREEN

Apare automat când kill switch e activ sau API-uri sunt down. Mesaje clare: "CRITICAL — Multiple systems down". Corect implementat.

🔴 API Health grid nu diferențiază criticitate

Toate sursele API sunt afișate egal. Dar MEXC DOWN = nu poți tranzacționa. CoinGecko DOWN = pierzi price feed secundar. Impactul e total diferit, dar vizual arată la fel.

Fix: Marchează sursele critice (MEXC, Supabase) cu un indicator "CRITICAL PATH". Când una critică e DOWN, evidențiază-o cu border roșu pulsant, nu doar un dot.

🟡 SSE connection status nu e evident pe mobile

Indicatorul "SSE LIVE" e un dot mic de 6px în header. Pe mobile, utilizatorul nu știe dacă datele sunt live sau stale.

Fix: Pe mobile, adaugă un banner subtil la top: "🔴 DISCONNECTED — data may be stale" când SSE nu e connected. Pe desktop, status-ul actual e ok.

Agent Orchestration Controls

✅ BUN: Cockpit Panic Button cu double-confirm

Pattern de armare (click 1 = arm, 4s window, click 2 = confirm) e sigur și previne accidental engagement. Poziționare fixă top-right e corectă.

🟡 Lipsa unui "action feedback" vizual persistent

Când execuți un command (e.g. "Collect Sentiment"), rezultatul apare doar în terminal. Dacă terminalul e collapsed sau user-ul nu se uită acolo, nu primești feedback.

Fix: Adaugă un toast/notification system: "✅ Sentiment collected (230ms)" care apare 3s overlay, indiferent de poziția de scroll.

🟡 Time-to-diagnosis prea mare

Când ceva e broken, user-ul trebuie să: (1) vadă alert bar, (2) scrolleze la API Health, (3) identifice sursa DOWN, (4) scrolleze la terminal, (5) caute eroarea relevantă. 5 pași, ~30 secunde.

Fix: Când alertLevel !== GREEN, inserează automat un "Diagnosis Panel" sub header care listează: ce e down, de când, last error message, și buton de remediere sugerat. Target: 1 click → diagnostic complet.

Logs & Debug

✅ BUN: Dual terminal system

Live Terminal (comenzi + rezultate) + Live Console (backend logs) = separare clară între acțiunile user-ului și event-urile sistemului.

✅ BUN: TerminalOverlay cu tab-uri EXECUTION | NEURAL

Separarea log-urilor AI (reasoning, confidence) de log-urile de execuție (ordere, fill-uri) e o funcționalitate premium. Bine gândit.

ℹ️ Lipsă: Persistent log search

Nu poți căuta în log-uri un simbol specific sau un error message. Pe 500 linii de terminal, asta face debugging mult mai lent.

Fix: Adaugă search bar în terminal header cu highlight pe match-uri.

4. PROPUNERI DE OPTIMIZARE

4.1 Restructurare Status Page (Impact: MAXIM)

Transformă Status Page din "lista de 15 secțiuni" în "3 zone strategice":

ZONA 1 — CRITICAL (always visible, never collapsed):

Header sticky + Core Services Strip + Alert Bar + KPI strip (Equity, P&L, Win Rate, MaxDD, Mode). Max 200px înălțime.

ZONA 2 — OPERATIONS (main workspace):

Terminal + Command Center + Strategy Performance. Visible by default. Tab-based, nu stacked.

ZONA 3 — DEEP DIVE (collapsed by default):

Charts, API Health detail, System Resources, Syndicate Audit, History/Export. Accesibil, dar nu în cale.

4.2 Unificare Color System (Impact: MARE)

Definește un singur token set în globals.css. Fiecare pagină poate avea un accent override, dar core palette rămâne identic:

--surface, --surface-alt, --border, --text, --text-dim, --text-muted — IDENTICE pe toate paginile.

Acum: Dashboard definește C.bg: '#07080d', Cockpit C.bgDeep: '#05020d', Arena '#050609', globals --bg-primary: '#06040a'. 4 valori diferite pentru background.

4.3 Toast/Notification System (Impact: MEDIU)

Implementează un ToastProvider global în AppShell.tsx:

Poziționare: bottom-center pe desktop (deasupra BottomNav pe mobile). Auto-dismiss 4s. Tipuri: success/error/warning/info. Maximum 3 toasts vizibile simultan, stack vertical.

4.4 Quick Diagnosis Panel (Impact: MARE)

Când alertLevel !== 'GREEN', renderează sub header un panel compact:

"⚠️ MEXC DOWN (3m ago) — Last error: Connection timeout. [Retry] [Details]"

Reduce time-to-diagnosis de la ~30s la ~2s.

5. CE MAI TREBUIE ADĂUGAT

1. Activity Timeline / Event Feed

O linie temporală vizuală care arată: trade opened → trade closed → kill switch engaged → API recovered. Cronologic, nu fragmentat pe secțiuni. Esențial pentru post-mortem analysis.

2. Sound/Push Notifications pentru events critice

Când kill switch se activează sau un API critic cade, user-ul trebuie să fie pe pagină ca să vadă. Adaugă: (1) browser notification via Service Worker (deja ai SwRegister), (2) optional audio beep pe alert RED.

3. Shortcuts cheat sheet accesibil

Ai shortcuts în sidebar (R, P, A, C, S) + ⌘K, dar nu există un vizual care le listează. Adaugă "?" shortcut care deschide un overlay cu toate shortcut-urile. Standard în orice power-user tool.

4. Dark/Darker mode toggle

Background-ul actual e foarte dark (#06040a). Unii utilizatori preferă un dark mai "luminat" (#111827). Adaugă un toggle: VOID (actual) vs SLATE (mai deschis).

5. Settings page funcțional

Sidebar-ul are "⚙️ Settings" cu opacity: 0.5 și cursor: default. E un buton mort. Ori implementează-l (preferences, notifications, thresholds), ori scoate-l complet.

7. DRAGON PREMIUM — SPECIFICAȚII COMPLETE

Status Actual Dragon

DragonLogo.tsx: SVG pur, 120x120 viewBox, ~190 linii. Folosit în Sidebar (26-32px). Cristalografic gold, ochi roșii, breathing animation CSS (scale 1→1.02), nostril flicker, eye glow. Bevel filter SVG.

MEDIU Arată decent la 32px, dar la orice dimensiune >60px se vede low-poly și amateurish. Filtrele SVG (bevel, glow) adaugă blur care spală detaliul.

Direcție Premium — Ce Trebuie Schimbat

CONCEPT: Dragon Silhouette Luxury, nu low-poly crystal

Renunță la abordarea "crystalline facets" (poligoane vizibile). Premium = forme fluide, clean, cu detaliu subtil. Gândește-te la un logo de hedge fund, nu la un joc video.

DIMENSIUNE: Mărește în sidebar la 44px collapsed / 56px expanded. Pe mobile header (dacă adaugi): 40px. Viewbox: 200x200 pentru mai mult detaliu.

DESIGN DIRECTION:

— Siluetă minimalistă de dragon din profil (cap + coarne + gât), nu frontal. Profil = mai regal, mai heraldic.

— Linii clean, fără stroke vizibil. Gradient gold subtil (3 stops: #FFE08A → #DAA520 → #996515).

— Ochi: singur punct luminos roșu (#FF2D55), nu elipsă mare. Mic, intens, lethal.

— Coarne: 2 arce elegante, nu spike-uri drepte. Thinning la vârf.

— Gât: scame sugerate prin 3-4 arce paralele, nu poligoane individuale.

ANIMAȚII PREMIUM (CSS, nu JS canvas):

1. Breathing glow: filter: drop-shadow care pulsează de la 0 0 8px gold/0.2 la 0 0 20px gold/0.4 pe 4s ease-in-out. Subtil, nu flashy.

2. Eye pulse: Ochiul pulsează independent, offset de breathing cu 1.5s. opacity: 0.6 → 1.0, filter: drop-shadow(0 0 3px #FF2D55).

3. Hover state: La hover pe sidebar, dragonul face un micro-tilt (rotate 2deg) + glow-ul crește la 0 0 30px gold/0.5. Smooth 0.4s transition.

4. Ambient light: Background radial gradient subtil în spatele dragonului: radial-gradient(circle, rgba(218,165,32,0.08) 0%, transparent 70%). Dă senzația că dragonul emite lumină.

5. NO: Fără 3D transform, fără perspective, fără animații de foc, fără particule. Premium = restrained power.

INTEGRARE UI:

— Sidebar collapsed: dragon centered, ambient light subtle. Singurul element vizual din sidebar care "trăiește".

— Sidebar expanded: dragon + "TRADE AI" text la dreapta. Dragon la stânga, text la dreapta, aliniat vertical centru.

— Nu adăuga dragon pe paginile principale. Dragon = brand mark, nu decorație. Doar în sidebar/header.

— Z-index: dragon-ul NU trebuie să suprapună conținut operațional niciodată.

8. PLAN MOBILE 100% — INSTRUCȚIUNI CONCRETE

Status Actual Mobile

PARȚIAL IMPLEMENTAT globals.css are media queries pentru 768px, 480px, 360px. BottomNav apare pe mobile. Safe area insets sunt setate. Dar paginile individuale (mai ales Status Page) au inline styles care overridează responsive-ul.

Breakpoints Necesare

≥1200px Desktop full — sidebar expanded on hover, grids 3-4 coloane
1024-1199px Desktop compact — sidebar collapsed permanent, grids 2 coloane
769-1023px Tablet — no sidebar, top nav optional, grids 2 col, BottomNav activ
481-768px Phone large — single column, BottomNav, compact cards
360-480px Phone small — ultra-compact, KPI-uri pe 2 coloane, hide non-essential
≤359px Phone mini — single column everything, stat cards stacked

Reguli Critice Mobile

🔴 FIX 1: Inline styles overridează responsive CSS

Status Page (dashboard/page.tsx) are ~200 inline style={{}} cu valori fixe (fontSize, padding, grid-template-columns). CSS media queries din globals.css nu pot override inline styles.

Fix: Mută toate inline styles critice în CSS classes. Folosește CSS variables pentru sizing care se schimbă per breakpoint. Exemplu: .grid-4 class există deja în globals.css cu responsive — dar dashboard-ul redefinește grid inline.

🔴 FIX 2: Arena leaderboard — minWidth: 200px pe fighter name

Arena page: style={{ flex: 1, minWidth: 200 }} pe fighter name container + metrici cu minWidth: 60-80px. Pe un ecran de 375px, asta forțează horizontal overflow.

Fix: Pe ≤768px: fighter plate devine vertical stack. Name + status top, metrici grid 2×2 bottom. Remove all minWidth pe mobile.

🔴 FIX 3: Cockpit kill button overlap pe mobile

Kill switch: position: fixed; right: 18px; top: 18px;. Pe mobile cu header sticky + sidebar collapsat, kill button suprapune header-ul sau dispare sub safe area.

Fix: Pe ≤768px: mută kill button ÎN header-ul cockpit-ului, lângă titlu. Sau fă-l parte din BottomNav ca emergency action.

🟡 FIX 4: Command Center buttons overflow

25+ butoane în flex-wrap. Pe 375px wide, încap ~3 butoane pe rând. User-ul scrollează mult.

Fix: Pe mobile: grupează comenzile în dropdown/accordion per categorie. Vizibil: Kill Switch + top 3 acțiuni frecvente. Rest: collapsate sub "More commands ▼".

🟡 FIX 5: Tables pe mobile

Globals.css setează table { display: block; overflow-x: auto } pe mobile. Corect, dar Syndicate Audit Trail și Decision History nu folosesc <table>, ci div-uri cu flex. Acestea nu beneficiază de regula CSS.

Fix: Pentru div-based tables: adaugă overflow-x: auto și -webkit-overflow-scrolling: touch pe container. Sau transformă-le în card-based layout pe mobile.

🟡 FIX 6: Touch targets prea mici

Globals.css setează button { min-height: 36px; min-width: 36px } pe ≤480px. Dar inline styles pe CmdBtn au padding: 6px 10px cu fontSize: 10px. Rezultatul: butoane de ~28px înălțime. Sub minimum 44px Apple HIG.

Fix: Pe mobile: toate butoanele interactive minimum 44×44px touch target (poți folosi padding invizibil). Tab buttons, filter buttons, expand/collapse — toate minim 44px.

FIX 7: Bottom padding pentru BottomNav

Page container are padding-bottom: 80px pe ≤768px. Dar Status page setează paddingBottom: 80 inline. Duplicate, dar funcțional. Cockpit și Arena au de asemenea paddingBottom: 80. Unificat ar fi mai curat în globals: @media (max-width: 768px) { .app-main { padding-bottom: 80px; } }

FIX 8: userScalable: false — dezactivează zoom pe mobil

layout.tsx: userScalable: false. Asta blochează pinch-to-zoom, ceea ce e un anti-pattern de accesibilitate WCAG. Utilizatorii cu probleme de vedere nu pot mări textul.

Fix: Schimbă la userScalable: true sau omite proprietatea. Dashboard-ul trebuie să funcționeze și cu zoom 200%.
