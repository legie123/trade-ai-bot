TECHNICAL DETAILS:

Why tsbuildinfo caused the build to fail:
  • Next.js incremental TypeScript compilation uses tsconfig.tsbuildinfo
  • File caches info about previous builds
  • When Docker context included stale tsbuildinfo, TypeScript reused stale cache
  • Cache referenced deleted route modules → "Cannot find module" errors
  • Despite actual source having no TypeScript errors
  
Solution:
  • Exclude tsbuildinfo files from Docker context
  • Each Docker build starts fresh with no stale cache
  • New .next/types/validator.ts generated only from current routes
